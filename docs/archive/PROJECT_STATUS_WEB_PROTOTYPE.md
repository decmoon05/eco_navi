# 🌿 Eco Navigation 프로젝트 현황 보고서 (웹 프로토타입 최종본)

**[본 문서는 모바일 AR 앱 개발에 앞서, 웹 프로토타입으로 구현된 모든 기능과 상태를 요약한 최종 개발 일지입니다.]**

## 1. 프로젝트 개요

'Eco Navigation'은 사용자가 친환경적인 이동 수단을 선택하도록 장려하고, 이동 시 발생하는 탄소 배출량을 계산하며, 그 절감량을 기록하여 성취감을 제공하는 **웹 기반 내비게이션 애플리케이션**입니다. 본 웹 앱은 최종 목표인 모바일 AR 앱의 핵심 기능과 로직을 검증하는 **성공적인 프로토타입** 역할을 수행합니다.

## 2. 구현 완료된 핵심 기능

*   **경로 검색:** Tmap API를 활용하여 도보, 자전거, 버스, 지하철, 자동차, 전기차 등 다양한 이동 수단에 대한 경로를 탐색합니다.
*   **정교한 탄소 배출량 계산:** 각 이동 수단(자동차, 버스, 지하철, 전기차)별 특성을 반영한 상세 공식과 보정계수를 적용하여 탄소 배출량을 정밀하게 계산합니다.
*   **사용자 계정 시스템:** Express와 SQLite 기반의 백엔드 서버를 통해 회원가입, 로그인, 로그아웃 기능을 제공하며, JWT와 `bcrypt`를 사용하여 안전하게 인증 및 상태를 유지합니다.
*   **페이지 분리 및 접근 제어:** `react-router-dom`을 사용하여 로그인, 메인, 마이페이지, 랭킹, 상점, 리포트, 퀘스트 등 페이지를 분리하고, 로그인 상태에 따라 페이지 접근을 제어합니다.
*   **ECO 지갑 시스템:**
    *   **포인트 적립:** 친환경 경로 이용, 업적/퀘스트 달성 시 보너스 포인트를 적립합니다.
    *   **포인트 사용:** '상점' 페이지에서 누적된 포인트로 가상의 상품을 교환할 수 있습니다.
*   **사용자 동기부여 시스템:**
    *   **활동 기록 & 통계:** 모든 이동 기록을 서버에 영구적으로 저장하고, '마이페이지'에서 통계 정보를 시각적으로 제공합니다.
    *   **월간 목표:** 사용자가 직접 월간 탄소 절감 목표를 설정하고, 달성률을 게이지 바로 확인할 수 있습니다.
    *   **업적 시스템:** 특정 조건을 달성하면 보상과 함께 주어지는 영구적인 배지 시스템입니다.
    *   **퀘스트 시스템:** 주간/반복 등 주기적으로 참여 가능한 미션을 제공하고 보상을 지급합니다.
    *   **랭킹 시스템:** '명예의 전당' 페이지에서 누적 탄소 절감량 기준 상위 20명의 순위를 보여줍니다.
*   **모바일 최적화 및 디자인:**
    *   **반응형 디자인:** 모바일 화면에서도 UI가 깨지지 않고 자연스럽게 표시됩니다.
    *   **PWA:** 홈 화면 설치 및 오프라인 캐싱 등 앱과 유사한 경험을 제공합니다.
    *   **디자인 시스템:** 일관된 색상, 폰트, 컴포넌트(버튼, 카드) 스타일을 적용하여 사용자 경험을 개선했습니다.

## 3. 기술 스택

*   **Frontend:** React 18, TypeScript, React Router, Styled Components, Axios
*   **Backend:** Node.js, Express, SQLite3, bcrypt, jsonwebtoken, dotenv
*   **Build Tool:** Create React App
*   **외부 API:** Tmap API

## 4. 해결된 주요 버그

*   **백엔드 안정성:** `async/await`와 콜백 혼용, JWT 비밀 키 로딩 실패, 데이터베이스 쿼리 응답 누락, 문법 오류 등 서버를 중단시키던 다수의 치명적인 버그를 해결했습니다.
*   **논리 오류:** 자동차 보너스/절약량 표시 오류, 장거리 대중교통 검색 오류 등 기능의 논리적 결함들을 수정했습니다.
*   **UI 및 컴파일 오류:** 추천 검색어 목록 미표시, 컴포넌트 렌더링 오류, `import` 순서 및 모듈 누락으로 인한 컴파일 오류 등 사용자 경험과 개발 환경에 영향을 주던 문제들을 모두 해결했습니다.

## 5. 향후 과제 (모바일 앱 개발)

**웹 프로토타입 개발 단계는 성공적으로 완료되었습니다.** 다음 단계는 아래의 자산을 활용하여 본격적인 **모바일 AR 앱 개발**에 착수하는 것입니다.

*   **이식 대상 핵심 자산:**
    1.  **백엔드 서버:** 현재 구축된 서버와 데이터베이스는 모바일 앱에서도 그대로 재사용합니다.
    2.  **핵심 계산 로직:** `carbonCalculator.ts`의 모든 정교한 계산 로직을 모바일 앱의 언어(C# 또는 JavaScript)로 변환하여 이식합니다.
    3.  **UI/UX 설계:** 현재 웹 앱의 페이지 구성과 사용자 흐름을 모바일 앱의 기본 설계도로 사용합니다.
*   **모바일 앱에서 구현할 신규 기능:**
    1.  **AR 발자국 시각화:** 탄소 배출량을 AR로 시각화하는 핵심 기능을 구현합니다.
    2.  **푸시 알림:** 퀘스트 완료, 월간 목표 달성 등 주요 이벤트 발생 시 사용자에게 푸시 알림을 보냅니다.
    3.  **네이티브 기능 연동:** 스마트폰의 GPS, 카메라, 가속도계 등 네이티브 센서를 직접 활용하여 더 정확한 데이터를 수집하고 AR 경험을 향상시킵니다.
